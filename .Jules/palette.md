## 2026-02-23 - Accessibility Verification with Playwright & Web Components
**Learning:** When verifying accessibility improvements in custom web components (Shadow DOM), Playwright automatically pierces open shadow roots, simplifying locator strategies. However, verifying the component's internal state (like ARIA attributes on buttons inside shadow DOM) requires ensuring the custom element is fully defined and upgraded before interaction. Also, relying on `unpkg` for demos can be fragile due to module resolution issues; using `esm.sh` provides better compatibility for modern ES module workflows.
**Action:** Always wait for `customElements.whenDefined('my-component')` or use `page.wait_for_function` to ensure custom elements are ready before asserting on their shadow DOM content. Consider using `esm.sh` for reliable module loading in demos.
